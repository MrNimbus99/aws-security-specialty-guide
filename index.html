Quiz 1 — Sample Question
A security engineer needs to create custom metrics in CloudWatch based on specific patterns in CloudTrail logs, such as failed console login attempts. What is the most efficient approach?
  • Configure CloudTrail to send metrics directly to CloudWatch
  • Create a Lambda function to parse CloudTrail logs and publish custom metrics
  • ✓ Use CloudWatch Logs metric filters on CloudTrail logs
  • Use Athena to query CloudTrail logs and manually create metrics
Show Explanation
✓ Why correctCloudWatch Logs metric filters can parse log data and create custom metrics based on patterns. When CloudTrail logs are sent to CloudWatch Logs, metric filters can detect specific patterns (e.g., failed console logins) and automatically create metrics that can be used for alarms and dashboards.
✗ Why A is wrongCloudTrail does not send metrics directly to CloudWatch. CloudTrail delivers log files to S3 or streams events to CloudWatch Logs, but it does not natively publish CloudWatch metrics. You must use metric filters on CloudWatch Logs to extract metrics from CloudTrail events.
✗ Why B is wrongWhile a Lambda function can parse CloudTrail logs and publish custom metrics via the PutMetricData API, this requires writing and maintaining custom code. CloudWatch Logs metric filters provide this capability natively without custom development, making them more operationally efficient.
✗ Why D is wrongAthena is designed for ad-hoc querying of historical data stored in S3, not for continuous real-time metric creation. It does not produce CloudWatch metrics automatically and cannot drive ongoing alarms or dashboards without significant manual effort.
Quiz 2 — Sample Question
A company needs to send CloudWatch Logs from Lambda functions in multiple accounts to a central OpenSearch Service cluster for analysis. What is the most scalable architecture?
  • ✓ Use CloudWatch Logs subscription filters to stream logs to Kinesis Data Firehose, which delivers to OpenSearch
  • Configure each Lambda function to send logs directly to OpenSearch
  • Use CloudTrail to capture Lambda logs and send to OpenSearch
  • Export CloudWatch Logs to S3 and use Lambda to load into OpenSearch
Show Explanation
✓ Why correctCloudWatch Logs subscription filters can stream logs in real-time to Kinesis Data Firehose, which can deliver to OpenSearch Service. This provides a scalable, managed pipeline without custom code in Lambda functions. Firehose handles batching, retries, and delivery to OpenSearch.
✗ Why B is wrongHaving each Lambda function send logs directly to OpenSearch requires custom code in every function, increases function execution time, and doesn't scale. If OpenSearch is temporarily unavailable, logs could be lost.
✗ Why C is wrongCloudTrail captures Lambda invocation API calls (e.g., Invoke) but does not capture application-level logs written by Lambda functions. Lambda application logs are sent to CloudWatch Logs, not CloudTrail.
✗ Why D is wrongExporting CloudWatch Logs to S3 is a batch operation (not real-time) and requires a separate Lambda function to load data into OpenSearch. This adds latency and complexity compared to the streaming approach with subscription filters and Firehose.
Quiz 3 — Sample Question
A security engineer enabled CloudTrail in the us-east-1 region and confirmed that management events are being logged to the designated S3 bucket. However, S3 object-level API calls (such as GetObject and PutObject) for a specific bucket are not appearing in the trail. What is the most likely cause?
  • CloudTrail data events for S3 can only be delivered to a CloudWatch Logs log group, not to the trail's S3 bucket
  • ✓ S3 data events are not enabled in the CloudTrail trail configuration for that bucket
  • The S3 bucket being monitored is in a different region than the trail
  • S3 server access logging must be enabled before CloudTrail can capture data events
Show Explanation
✓ Why correct (B)S3 data events (object-level API calls like GetObject, PutObject, DeleteObject) must be explicitly enabled in the CloudTrail trail configuration. By default, trails only log management events. Data events must be configured separately using event selectors or advanced event selectors, and can be scoped to all S3 buckets or specific buckets. Without enabling data events for the target bucket, no object-level activity will appear.
✗ Why A is wrongCloudTrail data events are delivered to the same destinations as management events — the trail's S3 bucket and optionally a CloudWatch Logs log group. There is no separate destination requirement for data events; they flow through the trail like any other CloudTrail event.
✗ Why C is wrongCloudTrail trails can log data events for S3 buckets in any region. A trail configured as a multi-region trail or an organization trail captures events regardless of the bucket's region.
✗ Why D is wrongS3 server access logging and CloudTrail data events are completely independent mechanisms. Server access logging writes access records to another S3 bucket, while CloudTrail records API calls. Neither requires the other to function.
Quiz 4 — Sample Question
An organization wants to implement automated security assessments that run daily across all accounts, checking for unencrypted EBS volumes, public RDS snapshots, and open security groups. The solution should provide a compliance dashboard. Which combination is most appropriate? (Select TWO)
  • GuardDuty for configuration compliance
  • ✓ Config conformance packs for automated assessments
  • Macie for infrastructure scanning
  • ✓ Security Hub security standards for continuous compliance checking
  • CloudTrail for compliance monitoring
  • VPC Flow Logs for security group analysis
Show Explanation
✗ Why B and D are correctSecurity Hub security standards (such as AWS Foundational Security Best Practices and CIS Benchmarks) continuously evaluate resources against predefined controls—including checks for unencrypted EBS volumes, public RDS snapshots, and overly permissive security groups—and present results in a compliance dashboard. Config conformance packs bundle multiple Config rules with optional Systems Manager Automation remediation actions and can be deployed organization-wide via AWS Organizations, enabling daily automated assessments at scale. Together they cover detection, dashboarding, and remediation.
✗ Why A is wrongGuardDuty is a threat-detection service that analyzes VPC Flow Logs, DNS logs, and CloudTrail management events for malicious activity (e.g., compromised instances, credential exfiltration). It does not evaluate resource configuration settings such as encryption or security group rules.
✗ Why C is wrongMacie uses machine learning to discover and classify sensitive data stored in S3 buckets (PII, financial data, credentials). It does not assess infrastructure configuration compliance for EBS, RDS, or security groups.
✗ Why E is wrongCloudTrail records API calls (who did what, when, and from where) but does not evaluate whether the resulting resource state is compliant. It is an audit trail, not a compliance engine.
✗ Why F is wrongVPC Flow Logs capture network flow metadata (source/destination IPs, ports, protocol, accept/reject). They do not inspect security group rules or assess resource configuration compliance.
Quiz 5 — Sample Question
A security engineer detects a GuardDuty finding indicating command-and-control activity on an EC2 instance in a production VPC. The incident response plan requires immediate, automated isolation of the affected instance without disrupting other instances in the same subnet. Which Systems Manager capability is most appropriate?
  • Systems Manager Run Command to execute a shell script that modifies network ACLs
  • ✓ Systems Manager Automation documents with security group modification actions
  • Systems Manager State Manager to enforce a compliant security group association
  • Systems Manager Session Manager to manually log in and detach the ENI
Show Explanation
✓ Why correctSystems Manager Automation documents provide reusable, version-controlled runbooks that can programmatically replace an instance's security group with a quarantine security group (denying all traffic) as a single orchestrated step. Automation documents support parameters, conditional branching, error handling, and can be triggered automatically by EventBridge, making them ideal for repeatable, consistent incident isolation that minimizes blast radius.
✗ Why A is wrongRun Command can execute scripts on instances, but it requires the SSM Agent to be reachable on the target instance. If the instance is compromised or the agent is disabled, Run Command cannot execute. Additionally, modifying network ACLs affects the entire subnet, not just the compromised instance, which could disrupt healthy workloads.
✗ Why C is wrongState Manager enforces a desired configuration on a schedule or at regular intervals, but it is designed for drift remediation over time, not for immediate, event-driven incident isolation. It cannot be triggered on-demand by a security finding.
✗ Why D is wrongSession Manager provides interactive shell access, but manually logging in and detaching an ENI is a slow, manual process that doesn't scale across multiple instances and introduces human error. Automation documents execute the same isolation steps in seconds without manual intervention.
Quiz 6 — Sample Question
A security team at a financial services company receives an alert that a terminated employee's IAM credentials were used overnight. The team suspects data was exfiltrated from S3 buckets containing customer records. Which THREE log sources should the team analyze to determine what data was accessed and where it was sent? (Select THREE)
  • EC2 instance metadata
  • ✓ CloudTrail S3 data events
  • ✓ S3 server access logs
  • ✓ VPC Flow Logs showing outbound traffic
  • CloudFront distribution settings
  • IAM policy documents
Show Explanation
✓ Why correctCloudTrail S3 data events (B) log object-level API calls such as GetObject and PutObject, showing exactly which objects were accessed and by which principal. S3 server access logs (C) provide detailed request-level information including requester IP, request time, and HTTP status, which helps correlate with CloudTrail findings. VPC Flow Logs (D) capture network metadata for traffic leaving the VPC, revealing outbound data transfer volumes and destination IPs that may indicate exfiltration endpoints.
✗ Why A is wrongEC2 instance metadata provides information about the instance itself (instance ID, AMI, IAM role) but does not log data access activity. It is not a log source for tracking S3 data exfiltration.
✗ Why E is wrongCloudFront distribution settings define CDN configuration (origins, behaviors, caching) but do not record which S3 objects were accessed directly via the S3 API.
✗ Why F is wrongIAM policy documents define what actions a principal is allowed to perform, but they do not record actual API calls or data access events. They show permissions, not activity.
Quiz 7 — Sample Question
A security team discovers that 15 EC2 instances across three Availability Zones may have been compromised by the same malware. They need to capture forensic evidence — including volatile memory and disk snapshots — from all affected instances as quickly as possible while maintaining chain of custody. What approach is MOST efficient? (Select TWO)
  • ✓ Use Automated Forensics Orchestrator for Amazon EC2 to automate evidence isolation, capture, and storage
  • Create AMIs of each instance and analyze the images individually in a forensic account
  • Manually SSH into each instance and run forensic collection scripts one at a time
  • Terminate all instances and restore from the latest known-good backup
  • ✓ Use Systems Manager Run Command to execute forensic collection scripts across all instances simultaneously
  • Use CloudWatch Logs Insights to query instance memory contents remotely
Show Explanation
✓ Why correctSystems Manager Run Command can execute forensic collection scripts (memory dumps, disk imaging commands) across all 15 instances simultaneously without requiring SSH access — it uses the SSM Agent already installed on managed instances. Automated Forensics Orchestrator (AFO) for Amazon EC2 is an AWS solution that automates the entire forensic workflow: isolating compromised instances, capturing memory and disk evidence, and storing artifacts in S3 with integrity verification. Together they provide the fastest, most scalable approach for multi-instance forensic collection.
✗ Why B is wrongCreating AMIs captures disk state but misses volatile memory (running processes, active network connections, in-memory encryption keys). AMI creation is also slower per instance and does not automate the full forensic chain of custody. For forensics, you need both memory and disk artifacts.
✗ Why C is wrongManually connecting via SSH does not scale to 15 instances across 3 AZs. It is error-prone (inconsistent script execution), time-consuming, and requires network access that may be restricted during containment. SSM Run Command provides agentless-like execution through the SSM Agent without opening SSH ports.
✗ Why D is wrongTerminating instances and restoring from backup destroys all forensic evidence — volatile memory, running processes, and any attacker artifacts not present in the backup. Evidence preservation is a prerequisite for root cause analysis.
✗ Why F is wrongCloudWatch Logs Insights queries log data that has already been shipped to CloudWatch. It cannot capture instance memory contents, disk images, or other host-level forensic artifacts needed for a compromise investigation.
Quiz 8 — Sample Question
A retail company's CloudFront-hosted e-commerce site is experiencing inventory hoarding by automated bots that add items to carts but never complete purchases. The security team needs to block these bots while allowing legitimate customers and search engine crawlers to access the site. What is the MOST effective approach?
  • Create a WAF rate-based rule to block all IPs exceeding 100 requests per 5 minutes
  • Enable CloudFront geographic restrictions to block countries with the highest bot traffic
  • ✓ AWS WAF Bot Control managed rule group with targeted protection level for client fingerprinting
  • Configure CloudFront signed URLs to require authentication for all page requests
Show Explanation
✓ Why correctAWS WAF Bot Control with targeted protection level uses machine learning, browser fingerprinting (JavaScript challenges, TLS fingerprint analysis), and behavioral signals to distinguish between legitimate users, verified bots (such as search engine crawlers), and malicious automated bots. Targeted protection goes beyond simple rate limiting by analyzing client-side signals to identify sophisticated bots that mimic human behavior, making it the most effective approach for inventory hoarding bots.
✗ Why A is wrongRate-based rules limit requests per IP but cannot distinguish between a legitimate user browsing heavily and an automated bot. Bots that rotate IPs or operate below the threshold would bypass this control entirely, and it risks blocking legitimate high-traffic users during flash sales.
✗ Why B is wrongGeographic restrictions block all users from specified countries, impacting legitimate international customers. Bots can operate from any geography, including the company's primary customer markets, making geo-blocking ineffective against distributed bot networks.
✗ Why D is wrongSigned URLs are designed for restricting access to premium or private content, not for bot detection. Requiring signed URLs for all pages would break normal browsing workflows, prevent search engine indexing, and does not address the bot identification problem.
Quiz 9 — Sample Question

The security engineer selects Amazon CodeGuru Security to scan the code repository. Which THREE types of security issues does CodeGuru Security detect? (Select THREE)
  • ✓ Hardcoded secrets and credentials embedded in source code
  • Overly permissive IAM policies attached to deployment roles
  • ✓ SQL injection, command injection, and other code-level injection flaws
  • Misconfigured Amazon S3 bucket policies
  • ✓ Insecure cryptographic implementations and resource leaks
  • Unpatched operating system vulnerabilities on build servers
Show Explanation
✓ Why correctAmazon CodeGuru Security uses machine learning and automated reasoning to analyze application source code. It specifically detects: (1) hardcoded secrets such as passwords, API keys, and access tokens embedded in source code; (2) code-level injection flaws including SQL injection, command injection, and LDAP injection; and (3) insecure cryptographic implementations and resource leaks such as improper cipher initialization or unclosed connections. These are all code-level security issues that CodeGuru is purpose-built to find before deployment.

Quiz 10 — Sample Question
A security team needs AWS Network Firewall to decrypt and inspect outbound HTTPS traffic from application servers before it leaves the VPC. The team must ensure encrypted traffic to specific external APIs is inspected for data exfiltration patterns. What configuration is required?
  • ✓ Configure a CA certificate in AWS Certificate Manager (ACM), associate it with a TLS inspection configuration, and define an inspection scope to select which traffic to decrypt
  • Import individual server certificates for each destination domain and attach them directly to Network Firewall rule groups
  • Enable TLS inspection globally on the firewall with no additional certificate configuration required
  • Create an IPsec tunnel between Network Firewall and each destination to decrypt traffic at the endpoint
Show Explanation
✓ Why A is correctFor outbound TLS inspection, AWS Network Firewall acts as a man-in-the-middle proxy. You must provide a CA certificate (via ACM) that the firewall uses to dynamically generate server certificates for intercepted connections. The TLS inspection configuration defines the CA certificate and scope (which traffic to decrypt based on source/destination criteria). Without the CA certificate, the firewall cannot re-sign traffic after inspection.
✗ Why B is wrongIndividual server certificates are used for inbound TLS inspection (where you own the destination servers), not outbound. For outbound traffic, a CA certificate is needed so the firewall can dynamically generate certificates for arbitrary external domains.
✗ Why C is wrongTLS inspection cannot be enabled without certificate configuration. The firewall needs cryptographic material (a CA certificate for outbound, or server certificates for inbound) to terminate and re-establish TLS sessions.
✗ Why D is wrongIPsec is a VPN protocol for site-to-site encryption, not a mechanism for TLS traffic inspection. Network Firewall uses TLS termination and re-encryption, not IPsec tunnels.
Quiz 11 — Sample Question
A financial services company hosts a web application on EC2 instances behind an Application Load Balancer with Amazon CloudFront. A recent penetration test revealed SQL injection vulnerabilities. The security team wants to implement defense-in-depth to protect against SQL injection at multiple layers. Which THREE actions should the team take? (Select THREE)
  • Enable Amazon GuardDuty to detect and block SQL injection attempts in real time
  • ✓ Add an AWS WAF web ACL with a SQL injection match condition rule set on the CloudFront distribution
  • ✓ Refactor the application code to use parameterized queries for all database interactions
  • Enable VPC Flow Logs to capture and filter SQL injection payloads at the network layer
  • ✓ Implement server-side input validation and sanitization in the application layer
  • Configure Amazon Inspector to scan running EC2 instances for active SQL injection attacks
Show Explanation
✓ Why correct(B) AWS WAF with SQL injection match conditions inspects HTTP requests at the CloudFront edge and blocks malicious SQL patterns before they reach the origin. (C) Parameterized queries (prepared statements) prevent SQL injection at the database layer by separating SQL logic from user-supplied input, making it impossible for injected code to alter query structure. (E) Server-side input validation provides an additional application-layer defense by rejecting or sanitizing unexpected characters before they reach database logic. Together these three controls provide defense-in-depth across edge, application, and database layers.
✗ Why A is wrongAmazon GuardDuty is a threat-detection service that analyzes VPC Flow Logs, DNS logs, and CloudTrail events for suspicious activity. It does not inspect HTTP request payloads and cannot detect or block SQL injection attempts.
✗ Why D is wrongVPC Flow Logs capture IP-level metadata (source/destination IP, ports, protocol, bytes transferred). They do not inspect packet payloads, so they cannot detect or filter SQL injection strings embedded in HTTP requests.
✗ Why F is wrongAmazon Inspector scans for known software vulnerabilities (CVEs) and network reachability issues on EC2 instances. It does not monitor live application traffic or detect active SQL injection attacks.
Quiz 12 — Sample Question
A multinational company uses Azure AD as its corporate identity provider and is adopting AWS. The security team needs to enable employees to access multiple AWS accounts using their existing Azure AD credentials through IAM Identity Center. Which federation protocol should be configured between Azure AD and IAM Identity Center?
  • OAuth 2.0 with client credentials grant
  • LDAP over SSL (LDAPS) direct binding
  • ✓ SAML 2.0 federation
  • OpenID Connect (OIDC) with authorization code flow
Show Explanation
✓ Why correctIAM Identity Center integrates with external identity providers exclusively through SAML 2.0 for federation. Azure AD is configured as a SAML 2.0 IdP, and IAM Identity Center acts as the service provider. Users authenticate with Azure AD, which issues a SAML assertion that IAM Identity Center consumes to grant access to AWS accounts.
✗ Why A is wrongOAuth 2.0 client credentials grant is used for machine-to-machine authentication, not for federating human user identities between an IdP and IAM Identity Center.
✗ Why B is wrongLDAPS is used for directory queries and bind authentication between applications and directory servers. IAM Identity Center does not use LDAPS for external IdP federation; it relies on SAML 2.0.
✗ Why D is wrongWhile IAM Identity Center uses OIDC internally for some features (such as issuing tokens to client applications), the external IdP connection for workforce federation is strictly SAML 2.0, not OIDC.
Quiz 13 — Sample Question
A media company stores assets in an S3 bucket in Account A. A partner company operating in Account B (a separate AWS Organization) needs to upload large media files to a specific prefix (/partner-uploads/) in the bucket. The security team requires that no long-term credentials are shared, access is scoped to only the upload prefix, and access can be revoked immediately if needed. Which approach meets ALL requirements?
  • Share Account A root credentials with the partner's development team over an encrypted channel
  • Disable S3 Block Public Access on the bucket and distribute pre-signed URLs that expire weekly
  • Create an IAM user in Account A with an inline policy scoped to the prefix and send the access keys to the partner
  • ✓ Configure an S3 bucket policy in Account A granting s3:PutObject to the partner's specific IAM role ARN scoped to the /partner-uploads/* prefix, and ensure the partner's Account B role has an identity-based policy allowing s3:PutObject on Account A's bucket
Show Explanation
✓ Why correct (D)An S3 bucket policy in Account A grants cross-account access to the partner's IAM role ARN, scoped to only the /partner-uploads/* prefix. The partner's role in Account B needs a matching identity-based policy. For cross-account S3 access, either a resource-based policy (bucket policy) or role assumption is required—both sides must explicitly allow. The bucket policy approach avoids creating IAM users, uses only temporary role credentials, and access can be revoked instantly by removing the bucket policy statement.
✗ Why A is wrongRoot credentials must never be shared. This violates the AWS Well-Architected Security Pillar and provides unrestricted account-level access far beyond the /partner-uploads/ prefix.
✗ Why B is wrongDisabling S3 Block Public Access exposes the bucket to the internet, not just the partner. Pre-signed URLs grant temporary access but are tied to the signing identity's permissions and become difficult to manage at scale. This also violates the immediate revocation requirement since distributed URLs remain valid until expiration.
✗ Why C is wrongCreating an IAM user with access keys creates long-term credentials, which directly violates the requirement that no long-term credentials be shared. Access keys must be rotated and can be leaked, whereas role-based temporary credentials expire automatically.
Quiz 14 — Sample Question
A company allows a team of developers to create IAM roles for their Lambda functions in a shared development account. The security team discovers that one developer created a role with AdministratorAccess. How should the security team prevent developers from granting permissions that exceed their own authorized scope?
  • ✓ Attach permission boundaries to the developers' IAM entities that cap the maximum permissions any role they create can have
  • Use an SCP to deny all IAM role creation actions for the development account
  • Grant developers PowerUserAccess, which automatically prevents privilege escalation on created roles
  • Create an AWS Config rule that deletes any IAM role created with more than ten policy statements
Show Explanation
✓ Why correct (A)Permission boundaries define the maximum permissions an IAM entity can have. When developers create roles, you can require them to attach a specific permission boundary via an IAM policy condition (iam:PermissionsBoundary). This allows developers to create roles freely but ensures those roles cannot exceed the boundary. This is the AWS-recommended approach for delegated administration without privilege escalation.
✗ Why B is wrongDenying all IAM role creation is too restrictive. The requirement is to let developers create roles while preventing escalation, not to block role creation entirely. SCPs operate at the account level and do not provide the granularity needed to cap the permission scope of created resources.
✗ Why C is wrongPowerUserAccess is a managed policy that grants broad permissions but excludes IAM and Organizations actions. It does not inherently prevent privilege escalation for users who ARE allowed to create IAM roles. The question specifies developers need role-creation ability, which PowerUserAccess does not grant.
✗ Why D is wrongAWS Config can detect non-compliant resources, but counting policy statements does not determine privilege level—a single-statement policy could grant AdministratorAccess. Auto-deletion is also reactive rather than preventive, and could disrupt running workloads that depend on the role.
Quiz 15 — Sample Question
A security team needs to provide 200 remote developers with encrypted access to resources inside a private VPC. Developers work from personal devices in multiple countries, and the solution must support certificate-based mutual authentication with split-tunnel routing so that only AWS-bound traffic uses the VPN. Which service meets these requirements?
  • AWS Site-to-Site VPN with a virtual private gateway
  • ✓ AWS Client VPN with mutual authentication and a split-tunnel configuration
  • AWS Direct Connect with a private virtual interface
  • An internet-facing Application Load Balancer with IP-based security group rules
Show Explanation
✓ Why correct (B)AWS Client VPN is an OpenVPN-based managed service designed for individual remote users connecting to a VPC. It supports mutual (certificate-based) authentication, Active Directory, and SAML-based authentication. Split-tunnel mode routes only AWS-destined traffic through the VPN tunnel, reducing bandwidth usage. This is the purpose-built solution for remote workforce access to private VPC resources.
✗ Why A is wrongSite-to-Site VPN connects an entire on-premises network to AWS via IPsec tunnels between a customer gateway device and a virtual private gateway (or Transit Gateway). It is not designed for individual remote users on personal devices — it requires a dedicated gateway appliance on the corporate network side.
✗ Why C is wrongAWS Direct Connect provides a dedicated physical network connection between an on-premises data center and AWS. It requires physical infrastructure at a colocation facility and is not suitable for remote developers working from personal devices across different countries.
✗ Why D is wrongAn internet-facing ALB exposes resources to the public internet. Security group IP-based rules cannot provide encrypted tunnel access, do not support mutual certificate authentication, and are impractical when developers have dynamic IP addresses across many countries.
Quiz 16 — Sample Question
A healthcare organization stores protected health information (PHI) in Amazon S3. A recent security audit requires the team to implement defense-in-depth controls that protect data at rest against unauthorized access, accidental deletion, and ransomware. Which TWO controls should the team implement to address the broadest combination of these threats? (Select TWO)
  • Enable S3 Transfer Acceleration to improve upload performance for large medical imaging files
  • ✓ Enable S3 Object Lock in compliance mode with a retention period matching the data retention requirement
  • ✓ Configure AWS Backup with Vault Lock in compliance mode and cross-Region copy rules
  • Configure Amazon CloudFront with origin access control for the S3 bucket
  • Enable S3 Intelligent-Tiering to automatically optimize storage class placement
  • Implement S3 Requester Pays to reduce organizational data transfer costs
Show Explanation
✓ Why correctS3 Object Lock in compliance mode provides WORM (Write Once Read Many) storage that prevents any user—including the root account—from deleting or modifying objects during the retention period, directly addressing accidental deletion and ransomware threats. AWS Backup with Vault Lock in compliance mode ensures backup recovery points cannot be deleted by anyone (including administrators) once the grace period expires, while cross-Region copy rules provide geographic redundancy. Together these controls protect both the primary data layer and the backup layer with immutability.
✗ Why 'S3 Transfer Acceleration' is wrongTransfer Acceleration uses Amazon CloudFront edge locations to speed up uploads over long distances. It is a performance optimization feature and provides no data-at-rest protection, encryption, or immutability.
✗ Why 'CloudFront with origin access control' is wrongOrigin access control restricts direct S3 access so users must go through CloudFront. This is an access-control mechanism for content delivery, not a data-at-rest protection control against deletion or ransomware.
✗ Why 'S3 Intelligent-Tiering' is wrongIntelligent-Tiering automatically moves objects between frequent and infrequent access tiers to optimize costs. It has no security, immutability, or data protection capability.
✗ Why 'Requester Pays' is wrongRequester Pays shifts data transfer and request costs to the requester. It is a billing configuration with no data-at-rest security function.
Quiz 17 — Sample Question
A junior engineer schedules a KMS customer managed key for deletion with a 7-day waiting period. The key encrypts production data across multiple services. The security lead is concerned the waiting period is too short. What should the security lead recommend?
  • Cancel the scheduled deletion and disable the key permanently instead of deleting it
  • ✓ Cancel the deletion, reschedule with a 30-day waiting period, and set up a CloudTrail-based CloudWatch alarm for any Decrypt calls referencing the key during the waiting period
  • Allow the 7-day deletion to proceed since KMS keys can be restored from backups after deletion completes
  • Cancel the deletion, delete all data encrypted by the key first, then reschedule deletion with a 7-day waiting period
Show Explanation
✓ Why correctExtending the waiting period to 30 days provides the maximum safety window. Monitoring CloudTrail for failed Decrypt calls using the key during the waiting period reveals any services or data that still depend on it. If dependencies are found, the deletion can be cancelled before it becomes irreversible.
✗ Why A is wrongDisabling the key permanently without deleting it leaves orphaned key metadata indefinitely and does not address whether the key is still needed. It avoids making a decision rather than safely completing one.
✗ Why C is wrongOnce a KMS key deletion completes, the key material is permanently destroyed and cannot be recovered. Any data encrypted solely with that key becomes permanently inaccessible. The 7-day window is too short for a production key used across multiple services.
✗ Why D is wrongDeleting all encrypted data first is impractical for a production key used across multiple services, and a 7-day window still does not address the original concern about insufficient safety margin.
Quiz 18 — Sample Question
A company uses AWS Secrets Manager to store Amazon RDS for MySQL database credentials. Applications retrieve credentials at runtime using the Secrets Manager API. During a recent automatic rotation, the application experienced brief connection failures because it was still using old credentials when the database password changed. A security engineer needs to prevent this issue in future rotations. What should the engineer do?
  • Configure the Secrets Manager rotation Lambda function to use a single-user rotation strategy with an extended rotation window
  • ✓ Configure the Secrets Manager rotation Lambda function to use an alternating-users rotation strategy with two sets of credentials
  • Increase the automatic rotation interval from 30 days to 90 days to reduce the frequency of disruptions
  • Cache the database credentials in the application's environment variables and restart the application after each rotation
Show Explanation
✓ Why correctThe alternating-users rotation strategy maintains two sets of credentials (using AWSCURRENT and AWSPREVIOUS staging labels). During rotation, a new set of credentials is created while the old set remains valid, preventing connection failures. This is the recommended approach for zero-downtime rotation.
✗ Why A is wrongThe single-user rotation strategy changes the password on the existing database user. If the application holds a cached copy of the old credentials, connections fail during the brief window between when the password changes and the application retrieves the new secret. An extended rotation window does not eliminate this gap.
